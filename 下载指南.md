# COMAC-LocalAI-Windows 下载与部署指南

## 方法一：直接拷贝（推荐，用于内网/离线机器）

### 第一步：下载项目源码

在**有网设备**上执行：

```bash
# 克隆仓库
git clone https://github.com/kogamishinyajerry-ops/COMAC-LocalAI-Windows.git

# 进入目录
cd COMAC-LocalAI-Windows
```

### 第二步：下载 Ollama 便携版（约 2GB）

下载地址：https://github.com/ollama/ollama/releases/latest

下载文件：`ollama-windows-amd64.zip`

放置到项目根目录的 `tools/` 目录下：

```
COMAC-LocalAI-Windows/
└── tools/
    └── ollama-windows-amd64.zip   ← 放入此文件
```

### 第三步：下载 Python 安装程序（约 25MB）

下载地址：https://www.python.org/ftp/python/3.11.8/python-3.11.8-amd64.exe

放置到项目根目录的 `tools/` 目录下：

```
COMAC-LocalAI-Windows/
└── tools/
    ├── ollama-windows-amd64.zip
    └── python-3.11.8-amd64.exe   ← 放入此文件
```

### 第四步：下载 LLM 模型（可选，约 4-5GB）

**有网设备**运行：

```bash
# 方法1：使用项目提供的下载脚本
.\download-model.bat

# 方法2：手动从 HuggingFace 下载
# qwen2.5-7b-instruct-q4_K_M 模型：
# https://huggingface.co/Qwen/Qwen2.5-7B-Instruct-GGUF/tree/main
# 下载 qwen2.5-7b-instruct-q4_k_m.gguf（约 4.4GB）
```

放置到 `ollama-models/` 目录下：

```
COMAC-LocalAI-Windows/
└── ollama-models/
    └── qwen2.5-7b-instruct-q4_k_m.gguf   ← 放入此文件
```

### 第五步：拷贝到移动硬盘

将整个 `COMAC-LocalAI-Windows` 文件夹拷贝到移动硬盘或内网机器。

### 第六步：运行初始化（在内网机器上）

双击运行 `setup.bat`，脚本会自动：
- 安装 Python 3.11.8（如需要）
- 创建 .venv 并安装依赖
- 解压 Ollama 到 `tools/ollama/`
- 静默安装 VC++ Redistributable
- 验证环境就绪

### 第七步：启动服务

双击运行 `start.bat`，等待 Ollama 服务启动后，打开浏览器访问：

```
http://localhost:7860
```

## 方法二：下载预构建安装包（推荐，用于快速部署）

### 第一步：下载安装包

在**有网设备**下载（139MB）：

```
https://github.com/kogamishinyajerry-ops/COMAC-LocalAI-Windows/releases
```

找到：`COMAC-LocalAI-Setup-1.0-CPU.exe`

### 第二步：拷贝安装包

将 `.exe` 文件拷贝到内网机器。

### 第三步：安装

双击运行安装包，按向导完成安装。

### 第四步：下载模型（内网机器需手动复制）

在内网机器上，需要手动将 `ollama-models/` 目录（含 GGUF 模型文件）复制到安装目录。

## 方法三：使用预构建 EXE（最小化方案）

仅分发 `dist/COMAC-LocalAI.exe`（105MB）+ Ollama（约 2GB）。

适用于已有 Python 环境的内网机器。

|------|----------|----------|
| 内存 | 8 GB | 16 GB+ |
| 硬盘 | 10 GB 可用空间 | 20 GB+ |

## 故障排查

### 提示 "Python 未找到"
→ 运行 `pre-deploy.bat` 或手动安装 Python 3.11+

### 提示 "Ollama 服务启动失败"
→ 检查 `tools/ollama/` 目录是否完整解压
→ 手动运行 `tools\ollama\ollama.exe serve` 查看错误信息

### Gradio 界面打不开
→ 确认端口 7860 未被占用
→ 检查 `logs/ollama.log` 中的错误信息

### 模型推理速度慢
→ 这是 CPU 推理的正常表现
→ 16GB 内存机器建议使用 4-bit 量化模型（qwen2.5-7b-q4_K_M）
