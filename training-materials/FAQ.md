# 常见问题解答 (FAQ)

**版本：** v1.0
**日期：** 2026年5月

---

# 一、基础问题

## Q1: 如何访问平台？

**A:** 在浏览器地址栏输入 `http://localhost:7860` 即可访问平台。
如果是在公司内网其他机器访问，请联系管理员获取正确的访问地址。

## Q2: 需要安装什么软件吗？

**A:** 作为Web应用，平台不需要安装任何软件。
只需使用现代浏览器（Chrome、Firefox、Edge、Safari最新版本）即可。
首次访问可能会有加载较慢的情况，请耐心等待。

## Q3: 支持哪些文件格式？

**A:** 平台支持以下格式：

| 类型 | 格式 |
|------|------|
| Word | .docx, .doc |
| PDF | .pdf |
| PowerPoint | .pptx, .ppt |
| Excel | .xlsx, .xls |
| 文本 | .txt |

## Q4: 有文件大小限制吗？

**A:** 有的。单文件大小限制为100MB。
超出限制时请拆分文件或压缩后上传。

## Q5: 是否需要注册账号？

**A:** 不需要。平台采用简化的访问方式，直接访问即可使用。

## Q6: 平台收费吗？

**A:** 不收费。平台使用开源组件搭建，无软件授权费用。

# 二、功能问题

## Q10: 为什么摘要生成很慢？

**A:** 摘要生成速度取决于：

1. **文档长度**：文档越长，处理时间越长
2. **服务器负载**：多人同时使用时可能较慢
3. **模型状态**：首次运行时需要加载模型，稍慢

**优化建议**：
- 适当拆分大型文档
- 选择非高峰期使用
- 耐心等待，首次加载后会有缓存

## Q11: 格式转换失败怎么办？

**A:** 转换失败可能原因及解决方案：

| 错误类型 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 格式不支持 | 选择了不支持的转换类型 | 查看支持的转换类型列表 |
| 文件损坏 | 原文件已损坏 | 重新保存原文件后上传 |
| 转换超时 | 文件过大或服务器忙 | 等待后重试，或拆分文件 |
| 服务异常 | LibreOffice转换引擎异常 | 联系管理员检查服务 |

## Q12: 批量处理有限制吗？

**A:** 批量处理有以下限制：

- 单次批量处理文件数：最多20个
- 单个文件大小：不超过100MB
- 总处理时间：不超过5分钟

超出限制时，系统会提示错误，请分批处理。

## Q13: 可以同时处理多个文件吗？

**A:** 可以。使用批量处理功能，最多可同时处理20个文件。

## Q14: 处理后的文件保存在哪里？

**A:** 处理结果会在浏览器中显示，您需要手动下载保存到本地。
临时文件会在处理后自动清理，不会在服务器上保留。

## Q15: 模板可以自定义吗？

**A:** 可以的。联系管理员添加自定义模板。
自定义模板需要使用Jinja2语法编写。

## Q16: 敏感信息检测能拦截文件外发吗？

**A:** 不能。敏感信息检测只是提醒功能，不会拦截或阻止任何操作。
如需严格管控，请结合其他安全措施。

# 三、故障排查

## Q20: 页面打不开怎么办？

**A:** 请按以下步骤排查：

1. **检查网络**：确保电脑已连接公司内网
2. **检查地址**：确认输入的地址正确（`http://localhost:7860`）
3. **检查服务**：联系管理员确认平台服务已启动
4. **清除缓存**：尝试清除浏览器缓存后重新访问
5. **更换浏览器**：尝试使用其他浏览器访问

## Q21: 文件上传失败？

**A:** 文件上传失败可能原因：

1. **文件过大**：超过100MB限制，请拆分文件
2. **格式不支持**：确认文件格式在支持列表中
3. **网络中断**：检查网络连接后重试
4. **浏览器问题**：尝试清除缓存或更换浏览器

## Q22: 处理结果为空？

**A:** 如果处理结果为空，可能原因：

1. **文档内容为空**：确认原文件有实际内容
2. **PDF是扫描件**：扫描的PDF没有文字层，需要OCR处理
3. **编码问题**：部分特殊编码的文档可能识别失败

**解决方案**：
- 尝试将PDF转为图片后重新处理
- 联系管理员协助处理特殊文档

## Q23: 页面显示异常怎么办？

**A:** 可尝试以下解决方法：

1. 刷新页面（F5）
2. 清除浏览器缓存
3. 尝试其他浏览器
4. 重新登录（关闭并重新打开页面）

## Q24: 复制结果失败怎么办？

**A:** 如复制结果失败，可尝试：

1. 使用鼠标手动选中内容后复制
2. 点击"下载"按钮保存为文件
3. 截图保存

## Q25: 下载的文件打不开？

**A:** 可能原因：

1. **格式问题**：下载的是TXT格式，需要用文本编辑器打开
2. **编码问题**：部分系统默认编码可能不兼容
3. **文件损坏**：重新处理并下载

**解决方案**：尝试使用专业软件（如Office）打开，或联系技术支持。

# 四、安全相关

## Q30: 我的文件会被保存吗？

**A:** 不会。平台采用以下安全策略：

- 用户上传的文件仅在处理过程中临时存储
- 处理完成后，临时文件会自动删除
- 不会将任何用户数据上传到外部服务器
- 平台运行日志中不记录文件内容

## Q31: 如何保证数据安全？

**A:** 平台从多个层面保障数据安全：

1. **网络安全**：完全运行在内网环境，与外部网络物理隔离
2. **存储安全**：临时文件使用后即删除，不持久化存储
3. **访问安全**：操作日志记录，便于审计追溯
4. **传输安全**：本地访问，数据不经过网络传输

## Q32: 管理员能看到我的文件吗？

**A:** 管理员无法查看用户的具体文件内容。
日志中仅记录操作类型、时间等元数据，不记录文件内容。

## Q33: 如何申请更高权限？

**A:** 平台当前版本为通用权限设计，无需特殊权限。
如有个性化需求，请联系管理员评估。

## Q34: 可以审计操作记录吗？

**A:** 可以的。系统会记录操作日志，包括：
- 操作类型（摘要/转换/填充等）
- 操作时间
- 文件类型

不记录具体文件和内容。

# 五、性能相关

## Q40: 支持多少人同时使用？

**A:** 当前版本建议单台服务器支持5-10人并发使用。
具体取决于服务器配置和使用场景。

## Q41: 可以处理多大的文件？

**A:** 单文件大小限制为100MB。
超出此限制可能会导致处理失败或超时。

## Q42: 响应时间一般多久？

**A:** 处理时间因功能而异：

| 功能 | 典型处理时间 |
|------|--------------|
| 文档摘要 | 10-30秒 |
| 格式转换 | 5-30秒 |
| 智能填充 | 5-15秒 |
| 批量处理 | 依文件数量 |

## Q43: 服务器资源不足怎么办？

**A:** 如遇性能下降：
1. 避开高峰期使用
2. 减少批量处理的文件数量
3. 联系管理员升级服务器资源
